# 需求说明

## 1. 项目名称
智约校园：多态空间调度与高并发活动综合平台

## 2. 项目目标
建设一个校园统一预约与活动平台，覆盖：
- 学术空间预约
- 体育设施预约
- 校园热门活动抢票/报名

平台不仅要实现完整功能闭环，还要重点解决：
- 不同资源类型的统一建模问题
- 时间与空间冲突校验问题
- 秒杀场景下的高并发稳定性问题
- 复杂订单状态一致性问题
- 可扩展规则配置问题

## 3. 用户角色
### 3.1 普通用户
- 注册 / 登录
- 浏览空间资源
- 发起预约
- 支付订单
- 取消订单
- 查看预约记录
- 报名 / 抢活动名额

### 3.2 管理员
- 维护资源信息
- 发布活动
- 配置预约规则
- 查看订单与活动报名数据
- 处理异常记录

## 4. 功能范围

### 4.1 多态空间建模与预订系统
#### 学术空间（会议室 / 讨论室）
- 资源为原子化，不可拆分
- 支持自定义连续时间段预约
- 每次有效预约前后自动加 5 分钟缓冲
- 缓冲期不计入用户预约时长，但必须参与冲突校验

#### 体育设施
- 资源支持组合化预约
- 时间按 1 小时离散槽位开放
- 支持单场地预约与多场地组合预约

### 4.2 校园热门活动聚合
- 支持活动发布、展示、报名、抢票
- 支持热门活动高峰抢票
- 必须保障高并发下系统不宕机
- 必须杜绝超发 / 超卖

### 4.3 复杂状态机与动态规则引擎
#### 订单状态
- 待支付/确认
- 已确认
- 已取消
- 已爽约

#### 规则引擎
支持配置多维规则，例如：
- 本科生限额 2 小时
- 信用分低于 80，额度减半
- 爽约扣减 10 分

要求规则可配置、可扩展，避免将复杂规则直接堆在 if-else 中。

## 5. 非功能要求

### 5.1 前端体验
- 响应式设计
- 支持桌面端与移动端
- 页面层级清晰，交互易用

### 5.2 无障碍
- 图片具备 alt
- 表单具备 label
- 色彩对比达标
- 支持键盘导航
- 参照 WCAG 2.2

### 5.3 性能
- LCP ≤ 2.5s
- INP ≤ 200ms
- CLS ≤ 0.1

### 5.4 安全
- 防 SQL 注入
- 防 XSS
- 防越权访问（BOLA / IDOR）
- 不在前端硬编码密钥
- 不明文暴露敏感信息

### 5.5 工程与交付
- 全量容器化
- 使用 Dockerfile 与 docker-compose.yml
- 配置自动化流水线
- 自动执行 Lint 与单元测试

## 6. 交付内容
- 可公网访问的演示链接
- 完整项目仓库
- README（架构设计、运行方式、测试账号）
- 架构与产品说明文档
- 3~5 分钟演示视频

## 7. 需求边界
- 允许使用 Mock 数据
- 不要求接入真实教务系统
- 技术栈不限
- 允许使用 AI 编程助手，但需要体现自主架构设计能力
