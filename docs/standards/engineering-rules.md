# Engineering Rules

## 目标

本仓库优先追求可交接、可审查、可维护，而不是一次性堆出很多文档。

## 基本规则

- 所有长期有效的知识必须写入仓库
- 需求、规则、设计决策、计划、进度分开存放
- 任何任务都应可从文档和 git 历史中恢复上下文
- 每次改动都应保持仓库处于干净、可继续工作的状态

## 文档规则

- `reference/` 只放参考资料和原始输入
- 正式规范、计划、进度、决策不放在 `reference/`
- 同一份信息只保留一个主维护位置，其他地方只做链接
- 重要设计选择写入 `adr/`

## Agent 协作规则

- 开始前先读 `progress/agent-progress.md`
- 开始前先读 `plans/feature-list.json`
- 只选择一个未完成的高优先级任务推进
- 完成后再修改任务状态，不允许先标完成
- 如果反复出现同类错误，应补规则或补模板，而不是只在聊天中提醒

## 质量规则

- 小步提交
- 提交前做最小必要自检
- 不在未说明的情况下重写既有参考资料
- 发现结构不清时，优先补导航和边界说明
