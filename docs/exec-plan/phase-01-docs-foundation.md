# Phase 01: Docs Foundation

## 阶段目标

把现有参考资料整理为可持续维护的 docs 系统基础层。

## 输入资料

- `docs/reference/README_说明.md`
- `docs/reference/01_需求说明.md`
- `docs/reference/02_规则说明.md`
- `docs/reference/03_系统架构图.md`
- `docs/reference/04_用户流程图.md`
- `docs/reference/05_订单状态机图.md`
- `docs/reference/06_ER图_数据库关系图.md`

## 输出物

- 稳定的目录结构
- 文档导航
- 任务与进度机制
- 首轮正式信息架构方案

## 任务拆解

1. 盘点现有 reference 文档的定位和缺口
2. 设计正式文档的信息架构
3. 决定哪些内容直接复用，哪些内容需要重写
4. 为下一阶段站点化或发布化准备结构

## 风险

- 参考文档可能存在信息重叠
- 图示类文档可能需要转换格式
- 业务边界和规则定义可能仍不完整

## 完成定义

- 每份 reference 文档都有明确用途
- 正式 docs 信息架构有文档化说明
- 下一阶段工作可直接按计划推进
