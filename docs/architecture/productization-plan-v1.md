# Productization Plan V1

## 目标

将当前“功能闭环已完成的演示站点”推进为“具备正式校方产品气质的校园服务平台”。

本阶段不推翻既有后端与前端路由结构，重点放在：

- 视觉系统重构
- 用户端与管理端信息架构重整
- 页面状态、文案和交互产品化
- 浏览器级回归与最终演示材料

## 产品定位

`CampusBook` 的定位不是宣传官网，也不是纯开发控制台，而是香港科技大学（广州）校内的预约与活动服务门户。

产品关键词：

- 可信
- 学院感
- 清晰
- 克制
- 高效

## 品牌与视觉基线

参考港科大（广州）官网主题 CSS，产品化阶段采用以下主色板：

- 主品牌蓝：`#002F6B`
- 内容蓝：`#1D4F92`
- 强调金：`#A88337`
- 辅助金：`#C8B388`
- 页面背景：`#F7F8FB`
- 卡片背景：`#FFFFFF`
- 主文字：`#142033`
- 次文字：`#61708A`
- 危险色：`#CC415C`

设计方向：

- 用户端：校方服务门户
- 管理端：深蓝框架 + 白底工作区
- 动效：少量、有意义、可感知，不做花哨堆叠
- 移动端：必须作为正式体验面考虑，而不是桌面端缩放结果

## 产品结构

### 用户端

- 首页
- 学术空间
- 体育设施
- 校园活动
- 我的订单

### 管理端

- 总览
- 资源管理
- 活动管理
- 规则管理
- 订单与关键记录

## 执行阶段

### `PUX-001` 设计系统与应用壳层

- 建立正式颜色、阴影、圆角、导航、按钮、表单、卡片基线
- 重构 AppShell、全局背景、顶部品牌区与通用区块
- 重做首页和管理首页的第一屏信息架构

### `PUX-002` 用户端产品化

- 重构首页入口区和用户主路径
- 重构学术空间、体育设施、活动、订单页面的列表和表单层
- 补齐成功态、错误态、空状态、登录引导

### `PUX-003` 管理端产品化

- 将当前 `/admin` 从“接口操作面板”升级为后台工作台
- 重构资源、活动、规则维护页的结构和密度
- 明确管理动作层级和反馈路径

### `PUX-004` 响应式与细节收尾

- 补齐移动端布局
- 调整文案与状态提示
- 统一边界态、禁用态、异常态和空态

### `PUX-005` 浏览器级回归与留证

- 建立真实浏览器回归基础
- 覆盖学生端与管理员端主路径
- 留下截图和验证证据，形成最终演示材料

## 验收标准

- 首页和关键业务页看起来像正式校方产品，不再像开发联调页
- 用户端和管理端共享统一视觉语言，但身份边界清晰
- 主要页面在桌面端和移动端都可正常使用
- 页面状态反馈完整，不依赖人工解释
- 浏览器级回归覆盖至少一条学生主链路和一条管理员主链路
